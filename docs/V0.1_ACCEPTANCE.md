# DeepReader v0.1 Acceptance Criteria

v0.1 is complete only when:

## Backend

- Text files can be ingested.
- EPUB files can be ingested.
- Ingested documents are split into deterministic stable records.
- Records are stored in SQLite.
- Each record has a stable_id, order_index, source_text, source_hash, and metadata.
- BM25 search works over source_text.
- Search results expose score, stable_id, source_text, and retrieval_method.
- FastAPI exposes document and search endpoints.

## Frontend

- User can view documents.
- User can inspect document records.
- User can run a search query.
- User can see ranked chunks, scores, stable IDs, and source text.

## Tests

The following tests must pass:

- text ingest
- EPUB ingest
- stable ID generation
- chunking
- BM25 ranking
- API document endpoints
- API search endpoint
- upload path safety

## Explicitly out of scope for v0.1

- LLM summaries
- embeddings
- hybrid search
- question answering
- citations
- Docker polish
- advanced UI styling
